Somnolence

The latest group project

This is currently the best large project i have worked on yet. I was in charge of Player movement, physics, a Dialogue System, and implemented Multiplayer functionalities into the game. It took a pretty long time to work on the phyics, because i wished to combine the already implemented Unity physics engine, (With some neccesary functionalities), and a new one, because the game itself is 2d, but the project was made in 3d. I fixed this by tweaking settings, and making extra scripts to halt any undesireable behavior from the physics objects. This paired with the fairly simplistic game mechanics, and implementation of multiple scripts to aid in debugging, testing and other minor issues, meant that i had a good amount variety in my work. Furthermore, i worked together with the Designers very closely, in order to create an experience with the least amount of bugs, and aided a lot in testing/fixing scripts for them. I tried very hard to make the Dialogue System as modular as possible, in order to make it as easy as possible for the designers to build what they needed. And lastly, i was also assigned to create the music, and with my albeit limited musical experience, i have still created some good music.

Shadering Engine

I got the bare bones framework of the Engine handed to me, having only loading 3d models in it. I added things like a functioning Ui, to change settings, light, post processing, moving the camera, and switching scenes. The biggest issue i faced was the Post Processing, just taking a while to incorporate properly into the code, due to struggling with Depth Render Buffers, but of course i figured it out. The next hardest propblem to face was actually switching scenes, where saving information did not properly work, so i made a custom scene loader in order to save data from previous scenes and thus effectively swap between them by loading various models. Overall i would say that this is one project i had a lot of fun with, especially finding problems to solve, because i went through multiple iterations of the "loading", including custom objects, making copies of always rendered items in order to save on "space" (which did not work). It was just great to finally get to work in C++, and actually have to learn how logic works in there, since before i mostly worked in C#, which has a way more distanced connection to the pc (As in; You can control less in c# due to the computer doing it automatically).
